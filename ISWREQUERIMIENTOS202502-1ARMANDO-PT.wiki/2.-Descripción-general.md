## 2.1 Perspectiva del producto

El sistema es una plataforma digital diseñada para gestionar la inscripción de cursos universitarios. Permitirá que los estudiantes consulten la oferta académica, revisen horarios, verifiquen la disponibilidad de cupos e inscriban asignaturas de manera simple y organizada desde dispositivos móviles o equipos de escritorio. La interfaz será clara y fácil de usar para facilitar el proceso académico.

Los usuarios podrán crear una cuenta, iniciar sesión y seleccionar los cursos disponibles según su programa académico. Cada inscripción quedará registrada en la base de datos central de la universidad, garantizando orden y trazabilidad en los procesos administrativos. La plataforma incluirá además la gestión de docentes y la administración de cursos por parte del personal autorizado.

El objetivo principal del sistema es reducir el tiempo de los procesos administrativos, disminuir errores en las inscripciones y agilizar la gestión académica. Con esto se busca ofrecer una experiencia más eficiente tanto para los estudiantes como para los funcionarios universitarios.

## 2.3 Características de los usuarios

A continuación, se describen las clases de usuarios que se anticipa utilizarán este producto. Cada clase se diferencia por la frecuencia de uso, el nivel de privilegios de acceso y el conjunto de funciones del sistema que emplean.

### 1. Estudiantes

•	**Frecuencia de uso**: Alta (especialmente durante los periodos de inscripción).

•       **Funciones utilizadas**: Consulta de cursos, inscripción, pago de matrícula y verificación de confirmación.

•	**Nivel técnico: Básico**; los usuarios deben poder navegar por interfaces web y realizar operaciones comunes en línea.

•	**Nivel de seguridad o privilegio**: Limitado; solo pueden acceder a su información personal y a los cursos disponibles.

•	**Nivel educativo**: Estudiantes universitarios.

•	**Experiencia**: Variable, dependiendo de su familiaridad con plataformas académicas.

•	**Descripción**: Es la clase de usuario más importante del sistema, ya que la plataforma está diseñada principalmente para facilitar el proceso de inscripción y consulta académica.

### 2. Administradores Académicos

•	**Frecuencia de uso**: Media a alta (principalmente antes y durante el inicio de los periodos académicos).

•	Funciones utilizadas: Creación, edición y eliminación de cursos; gestión de docentes, horarios y cupos; generación de reportes académicos y financieros.

•	**Nivel técnico**: Intermedio a alto; los usuarios deben conocer el sistema de gestión académica.

•	**Nivel de seguridad o privilegio**: Alto; tienen acceso total a la información de cursos, docentes e inscripciones.

•	**Nivel educativo**: Personal administrativo o coordinadores de programa.

•	**Experiencia**: Amplia experiencia en manejo de datos académicos y procesos institucionales.

•	**Descripción**: Clase esencial para garantizar la actualización y exactitud de la información dentro del sistema.

### 3. Docentes
•	Frecuencia de uso: Baja a media.

•	**Funciones utilizadas**: Visualización de cursos asignados, lista de estudiantes e información de horarios.

•	**Nivel técnico**: Básico a intermedio; deben manejar herramientas digitales para la gestión académica.

•	**Nivel de seguridad o privilegio**: Medio; acceso restringido a los datos de sus cursos asignados.

•	**Nivel educativo**: Profesores universitarios.

•	**Experiencia:** Habituales en el uso de plataformas académicas para registro y seguimiento de estudiantes.

•	**Descripción: **Clase relevante para el seguimiento académico, aunque con menor interacción que estudiantes y administradores.


### Clases de usuarios más importantes:
1.	Estudiantes, por ser los principales beneficiarios del sistema.

2.	Administradores Académicos, responsables de la correcta operación y gestión de datos.

Docentes se consideran una clase secundaria con requerimientos específicos y menos críticos para el funcionamiento general.
Cada clase de usuario tendrá una interfaz personalizada según su rol y nivel de acceso.


## 2.4 Ambiente de Operación


El sistema funcionará como una plataforma virtual para la inscripción de cursos universitarios y operará dentro del siguiente entorno tecnológico:


### Infraestructura tecnológica


• Computadores de escritorio y portátiles utilizados por estudiantes, docentes y personal administrativo.

• Dispositivos móviles como teléfonos y tabletas utilizados por los estudiantes para consultar horarios, cupos y realizar inscripciones.

• Servidores institucionales encargados de almacenar cursos, horarios, información académica y registros de pago.


### Conectividad


• Conexión a Internet estable para garantizar el acceso desde cualquier lugar.

• Sincronización en tiempo real entre la base de datos académica y las operaciones de inscripción, asignación de cupos y pagos.


### Interacción con hardware

• Equipos administrativos del área académica para gestión de cursos y profesores.

• No requiere dispositivos físicos especializados salvo lo tradicional en oficinas (impresoras para reportes o certificados si se necesitan).


### Condiciones ambientales


• Uso en entornos administrativos de la universidad y en espacios personales de los estudiantes.

•Funcionamiento continuo bajo condiciones estándar de temperatura e iluminación.


### Software operativo


• Plataforma accesible por navegadores modernos.

• Aplicación móvil para acceso rápido a horarios, cupos e inscripciones.

• Integración con un servicio de correo electrónico para enviar confirmaciones automáticas.

• Integración con pasarelas de pago en línea para los cursos de extensión.


### Guía de uso


• Instrucciones sobre registro, inicio de sesión, búsqueda de cursos y selección de horarios.

• Pasos para inscribirse y verificar cupos disponibles en tiempo real.

• Indicaciones para realizar pagos en línea cuando el curso lo requiera.

• Pasos para recibir y revisar la confirmación enviada al correo institucional o personal.


## 2.5 Restricciones y limitaciones

El desarrollo y funcionamiento del sistema de inscripción de cursos universitarios estará sujeto a las siguientes restricciones y limitaciones técnicas:

El sistema debe ser compatible con navegadores web modernos como Google Chrome, Microsoft Edge y Mozilla Firefox, garantizando su funcionamiento adecuado en entornos web estándar. La plataforma requerirá conexión estable a Internet para consultar horarios, verificar cupos en tiempo real, registrar inscripciones y procesar pagos en línea para cursos de extensión. El sistema será adaptable a diferentes dispositivos como computadores, tabletas y teléfonos móviles, aunque su rendimiento dependerá de la capacidad del hardware y la velocidad de conexión del usuario. La integración con servicios externos, como pasarelas de pago y sistemas de correo electrónicos institucionales, dependerá de la disponibilidad, funcionamiento y estabilidad de sus API. El acceso a las funcionalidades del sistema estará limitado por roles y permisos definidos, con el objetivo de mantener la seguridad de la información académica y evitar acciones no autorizadas. Además, la disponibilidad de cupos podrá verse afectada por la demanda alta durante los periodos de inscripción, lo que puede generar tiempos de espera o bloqueos temporales.


## 2.6 Documentación para el Usuario

La documentación para el usuario proporcionará toda la información necesaria para garantizar el uso adecuado de la Plataforma de Inscripción a Cursos Universitarios, tanto para los estudiantes como para los administradores académicos y el personal encargado del soporte.

Esta documentación incluirá los siguientes componentes:


### a) Manual de Usuario Final


Dirigido a estudiantes, docentes y administradores.

Incluirá instrucciones detalladas sobre:

• Cómo acceder a la plataforma mediante usuario institucional

• Cómo consultar cursos disponibles

• Cómo revisar horarios, cupos y prerequisitos

• Proceso de inscripción a cursos regulares y cursos de extensión

• Cómo descargar comprobantes y recibir confirmaciones por correo

• Cómo realizar pagos en línea para los cursos de extensión

• Recuperación de contraseña y actualización de datos personales


### b) Manual Técnico


Dirigido al personal de soporte o mantenimiento del sistema.


Contendrá información sobre:

• Instalación y configuración del sistema en servidores institucionales

• Requisitos mínimos de hardware y software del entorno operativo

• Configuración de la base de datos académica

• Parametrización de APIs para el envío de correos y pasarelas de pago

• Mantenimiento del sistema, monitoreo y detección de errores

• Procedimientos de respaldo, restauración y migración de información

• Políticas de actualización del sistema y control de versiones


### c) Guía Rápida


Documento breve para nuevos usuarios, donde se resumen las funciones esenciales:

• Cómo iniciar sesión con credenciales institucionales

• Cómo consultar cursos disponibles

• Cómo inscribirse en un curso

• Cómo revisar la confirmación de inscripción

• Cómo realizar pagos para cursos de extensión

• Cómo generar o verificar el comprobante de inscripción


### d) Centro de Ayuda Digital (en línea)


La plataforma contará con un módulo de soporte que incluirá:


• Preguntas frecuentes (FAQ) sobre inscripciones, horarios y cupos

• Tutoriales en vídeo explicando los procesos clave

• Guías interactivas de ayuda paso a paso

• Formulario de contacto para asistencia académica o técnica

• Chat de soporte automatizado para resolver consultas comunes


### e) Capacitación y soporte


Antes de la implementación del sistema, se ofrecerán sesiones de capacitación dirigidas a:


• Estudiantes: uso de la plataforma y procedimientos de inscripción

• Administradores académicos: gestión de cursos, docentes, horarios y cupos

• Personal técnico: mantenimiento, copias de seguridad y configuración

Posterior a la implementación, se contará con:

• Soporte técnico remoto para resolver incidencias

• Actualizaciones periódicas del sistema

• Recursos digitales permanentes para autoaprendizaje


## 2.7 Dependencias

| Tipo de Dependencia             | Descripción                                                                                             | Posible Impacto                                                                                       |
|---------------------------------|---------------------------------------------------------------------------------------------------------|--------------------------------------------------------------------------------------------------------- |
| Dependencia de hardware         | Se requiere disponibilidad de computadores, tabletas o dispositivos móviles con conexión a Internet.    | Si los equipos no cumplen requisitos mínimos, el sistema puede presentar lentitud o fallos de uso.   |
| Dependencia de Red              | El sistema depende de una conexión estable para cargar cursos, procesar pagos y enviar confirmaciones.  | Una mala conexión puede impedir inscripciones o generar errores en pagos y notificaciones.             |
| Dependencia de Base de Datos    | El sistema usará una base de datos SQL (PostgreSQL o MySQL) para almacenar usuarios, cursos y horarios. | Una configuración incorrecta puede causar pérdida de información o fallos en inscripciones.       |
| Dependencia de Software Terceros| Uso de frameworks y librerías externas como Django, Laravel, React o Bootstrap.                         | Actualizaciones o cambios pueden requerir ajustes en el sistema.                              |
| Dependencias de Sistemas de Pago| Integración con plataformas como PayU, Wompi o MercadoPago para cursos de extensión.                    | Cambios en las API pueden afectar el proceso de pagos en línea.                                          |
| Dependencia de API externas     | Posible conexión con servicios externos como sistemas académicos o plataformas institucionales.         | Cambios en las API pueden limitar sincronización con datos oficiales.                                    |
| Dependencia Organizacional      | Funcionamiento correcto depende de la capacitación del personal administrativo de la universidad.       | Errores en manejo del sistema pueden afectar inscripciones y actualización de información.               |
